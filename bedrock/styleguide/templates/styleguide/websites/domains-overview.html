Overview

If you’d like to create new Web content or functionality, please follow this guide to determine the right home for it in the Mozilla Web universe:

  1. mozilla.org is used for almost all sites
  2. new sites use sub-domains (example.mozilla.org)
  3. new pages on existing sites use sub-directories (mozilla.org/example).

Which to use depends on whether you’re creating new functionality or making use of existing functionality. Say you’re promoting the new breath-freshening feature in Firefox:

  • a new campaign that lets people submit videos of themselves talking about it would be a standalone site (fresh.mozilla.org); the purpose of the site determines the URL (addons.mozilla.org serves add-ons, support.mozilla.org provides support, etc.).
  • a static page about the feature would be added to the existing site (www.mozilla.org/firefox/fresh); the www signals that the site is for static Web pages

Note: It might make sense not to use mozilla.org on some occasions, though no other Mozilla domains should be used on public sites (mozilla.com, mozillafoundation.org, mozilla.net, etc.). We are in the process of phasing those out.
