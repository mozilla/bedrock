What your project should include

The SOS Fund has a very limited set of solid rules:

  • {% trans %} The software must be open source/free software, with a license that is OSI-certified and/or FSF-approved {% endtrans %}
  • The software must be actively maintained

How we’ll make our decision

{% trans %} We have a series of factors we consider when evaluating an application. {% endtrans %} {% trans %} For example: {% endtrans %}

  • How commonly used is the software?
  • {% trans %} Is the software network-facing or does it regularly process untrusted data? {% endtrans %}
  • {% trans %} How vital is the software to the continued functioning of the Internet or the Web? {% endtrans %}
  • {% trans %} Is the project known for something besides the code we are relying on? {% endtrans %}
  • {% trans %} Does the software depend on closed-source code, e.g. in a web service? {% endtrans %}
  • {% trans %} Are the software’s maintainers aware of and supportive of the application for support from the SOS fund? {% endtrans %}
  • Has the software been audited before? If so, when and how extensively? Was the audit made public? If so, where?
  • {% trans %} Does the software have existing corporate backing or involvement? {% endtrans %}

{% trans %} The answers to such questions are often not “yes” or “no”, but matters of degree, and so Mozilla will take the entire picture into account when assessing projects. {% endtrans %}

FAQ

{% trans %} We’ve been asked how this project compares to the Core Infrastructure Initiative of the Linux Foundation. {% endtrans %} {% trans %} Here’s a short answer: {% endtrans %}
{% trans %} We believe our model of support is different from and complementary to CII’s. {% endtrans %} {% trans %} We view CII as focused on necessary, deeper-dive investments into the core OS security infrastructure, like in OpenSSL. {% endtrans %} {% trans %} This is important work. {% endtrans %} {% trans %} Focusing on more point-in-time solutions, the SOS Fund’s audit and remediation methodology targets a different class of OSS projects with lower-hanging fruit security needs, using an open public-facing application form. {% endtrans %} {% trans %} To have substantial and lasting benefit in tackling such a significant issue as open source security, we need a broad range of solutions, including investment, audits, education, best practices, and a host of others. {% endtrans %} {% trans %} We believe the SOS Fund, alongside CII and other efforts, can help catalyze industry momentum to strengthen open source security. {% endtrans %}

{% trans %} If you have further questions, please feel free to contact us at sosfund@mozilla.com. {% endtrans %}
