{{ _('About') }}

{% trans %} Mozilla currently relies on a number of open source and free software projects. {% endtrans %} {% trans %} In some cases the software produced by these projects is incorporated into Mozilla products. {% endtrans %} {% trans %} In other cases we run it as part of our infrastructure, or Mozillians use it to get their jobs done. {% endtrans %} {% trans %} The Foundational Technology track is open to such projects who are in need of funds. {% endtrans %}

{{ _('What your application should include') }}

{% trans %} Applications should be jointly submitted by the leader of a project and an established Mozillian who will champion the award. {% endtrans %} {{ _('Your application needs to meet the following criteria:') }}

  • {{ _('Financial Request') }}

    {% trans %} Award requests should be for the amount of financial support needed to accomplish a clear and current project goal. {% endtrans %} {% trans %} Projects that don’t have a need for funds at this time can be considered when such a need arises. {% endtrans %} {% trans %} A $10,000 award request is just as valid as a $50,000 or $100,000 request. {% endtrans %} {% trans %} The money must be used to benefit the project, but it does not have to be used on things which themselves would directly benefit Mozilla. {% endtrans %}

  • {{ _('Project Origin') }}

    {% trans %} The project’s home can be anywhere in the world where we can make payment without undue burden. {% endtrans %} {% trans %} We have a reasonably strong preference for funding legal organizations rather than individuals representing projects, although exceptional circumstances could lead to some flexibility. {% endtrans %}

  • {{ _('Champion Involvement') }}

    {% trans %} The Mozillian champion may be (but does not need to be) a project contributor to the project they are championing. {% endtrans %} {% trans %} The Mozillian champion must vouch for the usefulness of the software, commit to monitoring the project, and report on the effectiveness of the award after the money is spent. {% endtrans %} {% trans %} They will also make themselves available to the application evaluation committee to help them assess the appropriateness of the award to the needs of the project. {% endtrans %}

{% trans %} We expect the vast majority of MOSS awards to be singular rather than regular — that is, support for doing a particular defined thing, rather than ongoing support for a project in general. {% endtrans %} {% trans %} However a previous award proposal, successful or unsuccessful, does not disqualify a project from applying again. {% endtrans %} {% trans %} The minimum award for the Foundational Technology track is $10,000, and the maximum is $250,000. {% endtrans %}

{{ _('How we’ll make our decision') }}

{% trans %} All criteria are indicative rather than determinative — that is to say, they will make us more or less likely to make an award, but none will guarantee a particular outcome. {% endtrans %}

{{ _('Selection criteria:') }}

  • {{ _('How reliant is Mozilla on the project’s technology?') }}
  • {{ _('Do others also rely on it outside of Mozilla products?') }}
  • {{ _('Is the technology unique?') }} {{ _('Are they doing something different?') }}
  • {{ _('What role does the project play in the open source ecosystem?') }}
  • {% trans %} What sort of reputation does the project have in general, if any? {% endtrans %}
    • {% trans %} This includes reputation in technical, inclusion and other areas {% endtrans %}
    • {% trans %} Is the project known for something besides the code we are relying on? {% endtrans %}
  • {{ _('Will this award make a significant impact on the project?') }}
  • {% trans %} Is the level of funding appropriate for the task to be accomplished? {% endtrans %}
  • {% trans %} Does the person (or group) who will receive the money have a track record of delivering? {% endtrans %}
  • {{ _('What does the champion have to say?') }} {{ _('Is the case compelling?') }}

{{ _('What a Mozilla champion does') }}

{% trans %} Applications need to be jointly submitted by the project and a Mozilla Champion. {% endtrans %} {% trans %} A Champion is an established member of the Mozilla community who knows the project concerned. {% endtrans %} {% trans %} Being a champion means you believe in the project and its impact. {% endtrans %} {% trans %} You believe that funds from a MOSS Award would make a meaningful difference in the success and effectiveness of the project. {% endtrans %} {% trans %} You believe the range of funds requested in the Award proposal is appropriate. {% endtrans %} {% trans %} And you believe that the project and the task fit the award criteria for the track you are applying for. {% endtrans %}

{{ _('A champion does a few different things:') }}

  • {% trans %} sponsors the project’s award proposal (it is akin to vouching for someone); {% endtrans %}
  • {% trans %} serves as the liaison between that project and Mozilla with regard to the award; {% endtrans %}
  • {% trans %} reviews the effectiveness of the award funds after an agreed upon period, to help Mozilla improve the effectiveness and impact of the MOSS program. {% endtrans %}

{% trans %} This review process might involve answering a set of standard assessment questions at the mid-point and at the end of the award period, and giving your written opinion of the overall success of the award. {% endtrans %} {% trans %} The time commitment should not be massive, but it will require thoughtfulness and honesty. {% endtrans %}

{{ _('FAQs') }}

{% trans %} What if my project doesn’t have a legal organizational home? {% endtrans %} {% trans %} I see this is disfavored, and exceptional circumstances are required. {% endtrans %} {% trans %} What might count as “exceptional”? {% endtrans %}
{% trans %} You tell us. {% endtrans %} {% trans %} If a project doesn’t have a legal organizational home, we will be handing a large sum of money to an individual with only limited accountability. {% endtrans %} {% trans %} This means a higher level of complexity and risk. {% endtrans %} {% trans %} So you need to describe why it’s reasonable and important for us to accept that additional risk. {% endtrans %} {% trans %} We will listen because Mozilla is a pretty flexible organization, but we can’t promise the result will be as you hope. {% endtrans %}
{{ _('What about tax?') }}
{% trans %} Recipients are responsible to determine the tax implications of receiving an award, based on their respective countries’ tax laws and compliance requirements. {% endtrans %} {% trans %} Amounts applied for should be sized such that any tax or other liability that will be incurred is accounted for. {% endtrans %} {% trans %} Mozilla reserves the right to use different payment mechanisms or tax characterizations for different awards in the MOSS program based on a number of factors specific to the award. {% endtrans %}
