{{ _('About MOSS') }}

{% trans %} Mozilla Open Source Support (MOSS) is Mozilla’s program for supporting the Open Source and Free Software movement, with a yearly budget of around $3 million. {% endtrans %} {% trans %} Mozilla was born out of and remains a part of this movement, and we prosper because of its technology and activism. {% endtrans %} {% trans %} And we know that open source software remains a key part of the Internet and is essential for the online life of choice, innovation and opportunity we seek to build. {% endtrans %}

{% trans %} The Mozilla Open Source Support program recognizes and celebrates open source projects that contribute to our work and the health of the Internet. {% endtrans %} {% trans %} It encompasses “giving back” to projects that Mozilla already relies on (“Foundational Technology”), “giving forward” to projects whose goals are in alignment with ours (“Mission Partners” and “Global Mission Partners: India”), and improving the security of the open source ecosystem (“Secure Open Source”). ”Global Mission Partners: India” is a pilot effort to focus resources on a vibrant regional open source community - in this case, in India. {% endtrans %}

{% trans %} MOSS applications are considered in batches every month. {% endtrans %} {% trans %} Missing a deadline may cause a delay in the consideration of your application. {% endtrans %} {% trans url1='https://wiki.mozilla.org/MOSS#Application_Deadlines' %} Information on the next deadline can be found on the MOSS wiki. {% endtrans %}

{{ _('Who decides?') }}

{% trans %} We have formed two selection committees to assess awards on the Foundation Technology and Mission Partners tracks. {% endtrans %} {% trans %} They are comprised partly of current, committed Mozillians — they bring a good working knowledge of Mozilla’s day- to-day activities and how various open source projects are used. {% endtrans %} {% trans %} There are also senior Mozilla Alumni — they are no longer actively involved with Mozilla on a day-to-day basis but have a deep understanding of our project and a different/outside perspective. {% endtrans %} {% trans %} Finally, there are other open source experts — they bring knowledge of the role of different projects within the open source ecosystem. {% endtrans %}
