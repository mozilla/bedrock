{{ _('What is Mozilla?') }}

{% trans url=url('firefox.new')%} Mozilla is a global, non-profit organization dedicated to making the Web better. We emphasize principle over profit, and believe that the Web is a shared public resource to be cared for, not a commodity to be sold. We work with a worldwide community to create open source products like Mozilla Firefox, and to innovate for the benefit of the individual and the betterment of the Web. The result is great products built by passionate people and better Web choices for everyone. For more information, visit mozilla.org/firefox. {% endtrans %}

{{ _('What is Mozilla Firefox ESR?') }}

{% trans url='https://wiki.mozilla.org/Enterprise/Firefox/ExtendedSupport:Proposal' %} Mozilla will offer an Extended Support Release (ESR) based on an official release of Firefox for desktop for use by organizations including schools, universities, businesses and others who need extended support for mass deployments. You can read more about the plan here. {% endtrans %}

{{ _('What does the Mozilla Firefox ESR life cycle look like?') }}

{% trans %} Releases will be maintained for approximately one year, with point releases containing security updates coinciding with regular Firefox releases. The ESR will also have a two cycle (12 week) overlap between the time of a new release and the end-of-life of the previous release to permit testing and certification prior to deploying a new version. {% endtrans %}

{{ high_res_img('img/firefox/organizations/release-overview.png', {'alt': _('Firefox ESR Release Overview'), 'width': '832', 'height': '384', 'id': 'release-overview'}) }}

{% trans %} Maintenance of each ESR, through point releases, is limited to high-risk/high-impact security vulnerabilities and in rare cases may also include off-schedule releases that address live security vulnerabilities. Backports of any functional enhancements and/or stability fixes are not in scope. {% endtrans %}

{{ _('Who should use Mozilla Firefox ESR?') }}

{% trans url=url('firefox') %} Mozilla Firefox ESR is meant for organizations that manage their client desktops, including schools, businesses and other instituitions that want to offer Firefox. Users who want to get the latest features, performance enhancements and technologies in their browsing experience should download Firefox for personal use, as these improvements will only be available to ESR users several development cycles after being made available in Firefox for desktop. If you’re using Firefox for personal use, you can download Firefox from mozilla.org/firefox. {% endtrans %}

{{ _('What does my organization need to do to use Mozilla Firefox ESR?') }}

{% trans url1='https://bugzilla.mozilla.org/enter_bug.cgi?product=Firefox', url2='https://wiki.mozilla.org/Enterprise:Bugs' %} Organizations that deploy Firefox ESR are requested to sign up for the Enterprise Working Group mailing list. They are also requested to test advance versions of Firefox and ESR Release Candidates. Users can file bugs for issues they may encounter here or follow known bugs here. {% endtrans %}

{{ _('How can I learn about the latest Mozilla Firefox ESR news?') }}

{% trans url1=url('firefox.organizations.organizations'), url2='https://wiki.mozilla.org/Enterprise' %} To learn about the latest Mozilla Firefox ESR news, you must be signed up for Mozilla’s Enterprise Working Group (EWG) mailing list. This is where Mozilla communicates the latest information about EWG, Firefox ESR and product updates. You can read more about EWG here. {% endtrans %}

{{ _('What is the Enterprise Working Group?') }}

{% trans url='https://wiki.mozilla.org/Enterprise' %} The Mozilla Enterprise User Working Group is where enterprise developers, IT staff, Firefox and Thunderbird developers discuss best practices for deploying Firefox and Thunderbird in the enterprise. You can learn more about the EWG here. {% endtrans %}

{{ _('Where can I get technical support for Mozilla Firefox ESR?') }}

{% trans url='https://support.mozilla.org/' %} Mozilla Firefox ESR support can be found through the EWG mailing list. As Firefox ESR is functionally identical to Firefox for desktop, users will be able to find relevant support articles here. {% endtrans %}

{{ _('How do I customize Firefox to meet my organization’s IT policy/requirements?') }}

{% trans url='https://addons.mozilla.org/en-US/firefox/addon/cck/' %} There are several methods that can be used to customize Firefox and the best place to start is the Enterprise Working Group or the Enterprise Wiki page. There are add-ons available, as well as third-party organizations who provide customizations services, and the EWG is a good place to discuss requirements if you’re just getting started. {% endtrans %}

{{ _('Is Mozilla Firefox ESR available in my language?') }}

{% trans url1='..', url2='https://mail.mozilla.org/listinfo/enterprise-de', url3='http://mozilla.jp/business/', url4='mailto:stormy@mozilla.com' %} Firefox ESR is available in every locale currently available for Firefox. Because of the precise needs of Firefox ESR, it’s important that anyone deploying the build be registered and participate in an EWG mailing list. The mailing list is only available in German, Japanese and English, but if you would like to volunteer to lead a Firefox ESR mailing list in your language, please get in touch. {% endtrans %}

{{ _('Is Mozilla Firefox ESR available for Android?') }}

{% trans %} No. Firefox ESR will only be offered for Windows, Mac and Linux for desktop computers. {% endtrans %}
